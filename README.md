## アプリの仕様について
"パーティーゲーム"として、アイスブレイク提案を行うwebアプリの作成を目指す。アプリの持つ要素として主に以下の二つを用意する。

- 話題提案ルーレット機能
- ゲーム提案機能

各機能についての仕様を以下に示す。

## 話題提案ルーレット

ランダムで話題を表示するものである。機能として以下をもつ.

- ルーレット機能
- 話題新規追加機能（"カテゴリー"と"テーマ"を入力する形式）

話題データについては事前に用意したものと新規でユーザーが追加していく形式とする。また、MySQLを使い、データベースとして管理する仕様とする（テーブルについては後述するが、各々のMySQLサーバーで現状管理している）

カテゴリーを使って、ジャンル別の話題ルーレットに変更できるような仕様を加えるかもしれない。

<u>ジャンルについて</u>

現状、"恋愛・人間関係"、"自己紹介・価値観系"、"日常・生活習慣"、"生活・ライフスタイル"、"仕事・勉強・工夫"、"趣味・エンタメ"、"食べ物・料理"、"子ども時代・過去エピソード"、"旅行・おでかけ"、"おもしろ発想"、"ガジェット・デジタル"としている。

<p class="info">myapp/data/topics.csvに使っているデータをおいておきます。</p>

### 話題ルーレット用データベース

以下に使用するデータベースの形式を記す。ユーザーが新規追加できるようにしたい。

|日本語|カラム|型|内容|
|---|---|---|---|
|テーマ名|theme|VARCHAR(100)|話題の内容|
|カテゴリ|category|VARCHAR(50)|カテゴリー名|

## ゲーム提案機能

アイスブレイクを提案するものであり、以下の機能をもつ。

- 検索機能
  - キーワード検索
  - 詳細設定検索
- ゲーム説明（テキストと画像スライド）

### 仕様について

- 検索機能

データベース内の対象のテーブルに対するSQLを結びつけることにより検索機能を実装する（キーワード検索はLIKE文などで）。

- 検索結果画面
ゲームアイコンとなるバナーを検索結果画面で表示し、それをクリックすることで詳細説明の画面に遷移するものとする

- テストプレイ
詳細説明画面より"テストプレイ"ボタンのクリックにより、遷移/表示させる説明スライド

- (新規追加機能)


### アイスブレイク用データベース
持つカラムについては以下のようにする
- ゲーム名：名前
- 人数：数字で表記
- 時間：数字（1~60の分表記にする予定）
- 道具：有or無で表記
- ジャンル：名前（文字列）
- 詳細：長々とした説明
- スライド（"テストプレイ"ボタンを押すと表示するスライド）：画像
- バナー：画像

↓　にMySQLで作成する想定のデータをまとめる

|日本語|カラム|型|内容|
|---|---|---|---|
|ゲーム名|name|VARCHAR(100)|ゲームのタイトル|
|人数|pyalers|INT|ゲームに必要な人数（だいたいの平均値）|
|時間|time|INT|ゲームにかかる時間|
|道具|items|BOOLEAN(TINYINT(1))|有or無で表記（詳細で使う小道具に関しては記載）|
|ジャンル|genre|VARCHAR(50)|ゲームのジャンル（コミュニケーション目的などで分ける）|
|詳細|detail|TEXT|これでいいのか不明|
|スライド|slides|VARCHAR(255)|テストプレイ時のスライド画像|
|バナー|banner_image|VARCHAR(255)|ゲーム名を記したアイコン|

-------------

両テーブルでのキー番号はいずれも入れた順に決定される仕様とする。
